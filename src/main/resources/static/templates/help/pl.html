Użytkownicy

Wszyscy użytkownicy aplikacji, są tutaj wylistowani, posortowani alfabetycznie lub pogrupowani po rodzinach.

Domyślne sortowanie może zostać zmienione w ustawieniach aplikacji przez administratora

Rodziny

Jedną z głównych funkcjonalnosci aplikacji jest grupwanie po rodzinach.
Dzięki temu, można o wiele szybciej przeglądać prezenty oraz zarządać listami członków rodziny.
Tworząć nową rodzinę, można dowolnie ją nazwać (w przypadku kiedy pozostawione puste, zostanie użyte twoje nazwisko). Podczas tworzenia, lub potem podczas edycji, możesz dodać innych użytkowników aplikacji, którzy jeszcze nie są dodani do żadnej rodziny.

Możesz też określić administratorów rodziny, którzy będą mogli dodawać i edytować prezenty do członków rodziny.
Aby stworzyć rodzinę, na ekranie listy użytkowników, kliknij guzik   , aby ją edytować , w tym samym miejscu będzie dostępny wtedy guzik  

Konto dziecka

Po stworzeniu rodziny, jako administrator rodziny będziesz mógł stworzyć konto dziecka, po kliknięciu na guzik {{'user.family.add.kid'|translate}}
Konto dziecka to spacjny rodzaj konta w aplikacji, na któr enie można się zalogować. Tylko administrator rodziny może dodawać i eydtować prezebty dla tego konta.

Tworząc konto dziecka , proszę okreś imię nazwisko oraz unikalny login ( aplikacja sprawdzi , czy jest wolny) . Ustaw również awatar konta ( opcjonalnie )

Po stworzeniu, konto dziecka można edytować po kliknięciu guzika w wierszu konta dziecka na ekranie listy użytkowników.

Gifts

In here all your gifts for currently browsed user

All gifts are grouped in categories and sorted by date. Newest gifts at top, old at the bottom.

Claimed gifts are additionally colored in orange with information who claimed on mouse over on claimed icon
(gift owner will not see this information).

Claimed gifts are shown at the bottom in their category Information like gift link , description and available search engines is shown for each row

Actions
Here are all actions available for gift. Depending if you are on your list or other users list, different actions are available.

- Mark gift as realised
- Edit gift
- Delete gift
- claim gift ( or remove claim)

Adding new gift

To add new gift, click Add newbutton on wish list view

If you are family administrator, you can add gifts on wish list of other family members

While creating new gift, only name field is required.
When adding gift, try to make it as a keyword, which will be used by search engines.

Other fields are optional but helpfull.

It's good to add new gift to category, and select which search engines will be visible for it

Claming gift

Each application user can claim other users gift. When claimed gift will be makred as claimed, with information who claimed it on mouse hover
If claimed, gift will look like this: Claimed gift

Search engines

Search engines can be defined by application administrators.Whenever this engine will be created while creating new gift, it's link will be shown next to gift link. When clicked search string will use gift name and search using that search engine.
For example if google search engine will be used for gift with name "MY_NAME", google will be used to search for https://www.google.com/search?q=MY_NAME

Gifts importing

It is also possible to import many gifts at once. Start by clicking button on gifts page.

Select previously filled in excel file (created based on downloaded template), and click Import button

All valid gifts from file will be imported for current user
(or family member, which gifts were currently browsed)

Application management

Application administrator has additional menu available on desktop version of app. Manage view can be accessed on mobile devices as well but only via direct link , not to obscure small mobile menu.
Please note that all properties have to be set on first visit , otherwise it's not guarantied that application will work correctly

{{'app.manage.language'|translate}}

Here you can set application default language.This list is dynamically populated by API from all available languages added in system.
If you want to add your language just copy and translate lang/messages_XX.properties and add it to application package. File can be manually added, but recommended is to add it to sources and perform mvn package.
Please note that in order for new language to be recognised it has to have at least main.language.name property

{{'app.manage.gift.age'|translate}}

{{'app.manage.gift.age.help '|translate}}

{{app.manage.url}}

This is application url, which will be used in mails and other forms of cumincation. It's recomended to set it on first visit

{{'user.sort.default'|translate}}

How user will be sorted by default on user list view

{{'app.manage.email'|translate}}

Base application email settings. This server will be used to send some basic mails to users, using application
After setting detials and saving , SMTP server will be tested if valid.

Search engines

Here you can define search engines which user will be able to use on their gifts
When defining new search engine it's crucial to all fields be valid.
Search engine name - name which will be shown on gift creation screen, and on mouse hover over icon
Search engine link - search query link. Gift name will be pasted at the end of it. Ussually it's in form : http://google.com/search?q=
Search icon - {{'app.manage.search.icon.help'|translate}}
