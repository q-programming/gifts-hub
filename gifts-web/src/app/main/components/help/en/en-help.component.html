About application

Welcome in Gifts Hub

Gifts hub is a place where you can share your gifts wish list with your friends and family. Every application member has his own list, to which can easily add, edit or remove gifts which he/she would like to receive
Application accounts can be created either with login password, or login using social media : facebook or google
In case of social media created account, all data like name, surname and photo will be automatically created

You can also download android application from here : cloud_download gifts-hub-1.0.0.apk
and install it on your phone ( allowing instalation from uknown sources option is requred)

Groups

Application permissions are based on groups - only people within group can view eachothers wish lists. Of corse you can be member of many groups at once
Thanks to this, you can very quickly browse and manage through other users wish lists.
When creating new group, you can name it as you want ( in case of empty name, your surname will be used ). While creating ( or while editing ) you can add other application users, by entering theirs email address or login
Every newly invited member will receive e-mail with link to confirm invitation.

To create new group on user list screen , click button

Editing group

To edit group, choose {{'user.group.edit.text'|translate}} from group menu.
In shown dialog , you can change group name and invite new members
To invite new memebrs , enter their logins or email addresses. If email owner doesn't have account in application , he will be invited to create one.

While editing group , you can also mark one members as group administrator. Group administrator can set other users as group admins, edit kids accounts and invite or kick users from group
It's not possible to remove last administrator

Kid account

After creating group, as it's administrator you can create kid account, after clicking option from group menu
Kid account is a special account type to which you cannot sign in ( and it doesn't has email assigned to it)
When creating kid account, defin name, surname and choose unique login ( application will check if it's unique).Set up avatar ( which is optional ) and it's ready

After creation, you can edit it by clicking button in kids row

Leaving group

If you wan to leave group, click button

When group admin is leaving group, and there is at least one group member ( which is not a kid ), the account will be automatically granted group admin In case when there are no valid accounts , group will be removed permamently

Adding gifts

Basing application functionality is adding gifts to your wish list. Apart from that any group member can add each other gifts to theirs list
After creating account there are no gifts.

They can be added by clicking button.

When creating new gift , only field name is mandatory
Creating name of gift, try to make it a keyword, as it will be used in search engines.

Other fields are optional , but very helpfull.

It's also good to add gift to one of categories ( or create new one ) and choose search engine

Gift properties

Short gift name. It will be used by search engines as keyword Gift description, where we can describe what kind of ift, color etc. Links to pages with gift. To add more links , click button Gift category. All gifts in that category will be shown together in one tab. You can choose one of categories,previously created by other users, or enter new one
If no category is given , gift will be placed in "Other" category Option available if we are on other users list.
When gift marked as hidden, it will be visible to all other users but list owner. User which added this gift , will be the only one who will be able to edit it, delete or mark as complete. Which search engines will be available for this gift. A bit more information about search engines here

Actions available on gifts

On gifts there are following actions available ( some options are not visible on other users lists) If gift is not yet claimed, you can claim it clicking this button, to notify other users that you want to realize this gift. In case we claimed gift, instead of this icon there will be this visible If we are author of gits, we can edit it here. In case we are on our list ( or we are author of normal or hidden gift )we can mark gift as realized.It will be then colored in gray and moved to "Realised" category .You can undo marking as done, by clicking action visible in same place Removing gift. If somebody already have reservatiot on this gift, he/she will be notified by gift removal by email

Filtering gifts

It's possible to filter gifts based on theirs categories for currently shown user.
To filter , click icon located in list header .
After clicking, extra select list will be shown , where you can choose one of existing categories.
If filter is active, icon will be changed to
To remove filtering , click clear button in filter category field.

Search engines

Search engines are defined by application administrator. When search engine will be marked while creating/editing gift, link to it will be then shown in gift row, next to any gifts links. After clicking, in new window search result will be shown using gift name as keyword
For example , if we use google search engine for gift with name "MY_NAME" , search result for
https://www.google.com/search?q=MY_NAME will be opened in new window.

Settings

Your account settings are located in menu under your avatar

On this screen you can add your unique avatar, change application display language and define if your list will be publicly visible.

Email notifications

To recieve email notifications with new and realised gifts you have to select to receive those. This setting will also be used to notify of any claimed deleted gifts
Newsletter will be sent every monday ( if there were any events concerning your account ) to email linked with your account (given while registering or shared with facebook/google accounts)

Public wish list


By clicking next to your public link, it will be copied to clipboard

Deleting account

You can permently remove your application account. This operation cannot be undone, all your gifts and claims will be removed together with your account
Remeber, this operation cannot be undone, so please use it with caution!

Application management

Presuming that users will use application according to its assumptions, administrator jobs shoudn't be hard

Application administrator have an extra option in menu.
Please remember to setup everything on first visit, or application might not work correctly

{{'app.manage.app.text'|translate}}

Define default application language. List will be automatically filled in according to all available languages installed
To add new application language , there are couple steps needed. First translate cotnents of gifts-hub\gifts-app\src\main\resources\templates\XX\ Add translations to lang/messages_XX.properties , and add portal translations gifts-hub\gifts-web\src\assets\i18n\XX
After whole operation build application mvn package in main application directory, and replace package on server Application link, which will be used to send email messages and other communications. It's recommended to set this parameter on first visit {{'app.manage.gift.age.help'|translate}} How users are sorted by default on user list view

{{'app.manage.search.engine'|translate}}

Here you can define all search engines, that users will be able to set on their gifts
When defining new search engine, it's vitial to correctly set up all fields
name that will be shown while creating new gift, and after hovering with mouse cursor over search engine icon search engine link. Name of gift will be pasted at the end of that link. Ussually the link is in format : http://google.com/search?q=

{{'app.manage.email.text'|translate}}

After setting up and saving, validity of configuration will be validated automatically
